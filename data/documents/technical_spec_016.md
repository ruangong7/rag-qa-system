# 镜像规范 

## 规范范围
本技术规范适用于 `billing-worker` 和 `RabbitMQ` 系统中的容器镜像机制。该规范定义了使用基础镜像（base image）的具体要求，包括必须实现的机制、参数以及验证信号。本规范不适用于客户服务脚本（customer support scripts）和人力资源审批流程（HR approval）。

## 兼容性表格
| 项目 | 描述 |
| --- | --- |
| **系统** | `billing-worker`, `RabbitMQ` |
| **基础镜像** | 必须使用非 root 用户（non-root user）运行 |
| **验证信号** | 软件物料清单（SBOM） |

## 客户端责任
- **MUST**：客户端在发布前必须测试基础镜像（base image），确保其符合规范。
- **SHOULD**：建议客户端使用自动化工具进行镜像扫描，以提高效率并减少人为错误。
- **MUST**：客户端必须确保镜像中包含软件物料清单（SBOM），以便于后续的安全审计和漏洞管理。

## 服务端责任
- **MUST**：服务端必须提供一个安全的基础镜像（base image），并且该镜像必须配置为使用非 root 用户（non-root user）运行。
- **SHOULD**：服务端应定期更新基础镜像，以修复已知的安全漏洞。
- **MUST**：服务端必须提供软件物料清单（SBOM），并在每次镜像更新时同步更新 SBOM。

## 监控信号
- **MUST**：监控系统必须能够检测到基础镜像（base image）的任何变更，并触发相应的告警。
- **SHOULD**：建议监控系统能够识别并报告镜像中的潜在安全风险，例如未使用的依赖项或过期的库。
- **MUST**：监控系统必须能够验证镜像是否由非 root 用户（non-root user）运行。

## 异常路径
- **MUST**：如果基础镜像（base image）未能通过安全检查，必须立即停止使用该镜像，并通知相关团队进行修复。
- **SHOULD**：建议在发现镜像存在安全问题时，启动应急响应计划，以最小化潜在的风险。
- **MUST**：如果镜像中的软件物料清单（SBOM）缺失或不完整，必须重新生成并更新 SBOM，然后重新发布镜像。

## 不适用范围
本规范不适用于客户服务脚本（customer support scripts）和人力资源审批流程（HR approval）。
