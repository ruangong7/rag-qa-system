# 服务等级目标

## 规范范围
本技术规范适用于 `identity-sdk` 系统中的服务等级目标（SLO）实现。该规范定义了 Tier-1 读取 API 的性能要求，包括必须实现的机制、参数以及验证信号。本规范不适用于法律保留政策（legal retention）和架构所有权（architecture ownership）。

## 兼容性表格
| 系统         | 是否适用 |
|--------------|----------|
| identity-sdk | 是       |
| CI Pipeline  | 是       |
| 法律保留     | 否       |
| 架构所有权   | 否       |

## 客户端责任
- **MUST**：客户端在调用 Tier-1 读取 API 时，必须确保请求格式正确。
- **SHOULD**：推荐客户端在接收到响应后进行数据校验，以确保数据完整性和一致性。
- **MUST**：客户端必须处理超时情况，并设置合理的重试策略。

## 服务端责任
- **MUST**：服务端必须保证 Tier-1 读取 API 的 P95 响应时间低于 500 毫秒（P95 below 500 ms）。
- **MUST**：服务端必须提供 99.9% 的可用性（99.9% availability）。
- **SHOULD**：推荐使用负载均衡器来分发请求，以提高系统的整体性能和稳定性。
- **MUST**：服务端必须记录所有请求和响应的时间戳，以便后续分析和监控。

## 监控信号
- **MUST**：系统必须收集并监控以下指标：
  - 请求延迟（Request Latency）
  - 请求成功率（Request Success Rate）
  - 服务可用性（Service Availability）

- **SHOULD**：推荐使用 Prometheus 或 OpenTelemetry 进行监控和报警配置。
- **MUST**：当 P95 响应时间超过 500 毫秒或可用性低于 99.9% 时，系统必须触发警报。

## 异常路径
- **MUST**：当 Tier-1 读取 API 出现异常时，服务端必须返回标准的 HTTP 错误码。
- **MUST**：对于超时错误，服务端必须返回 504 Gateway Timeout。
- **MUST**：对于内部服务器错误，服务端必须返回 500 Internal Server Error。
- **SHOULD**：推荐在日志中记录详细的错误信息，以便问题排查。

## 禁止事项
- **MUST NOT**：禁止在生产环境中禁用或绕过监控和报警机制。
- **MUST NOT**：禁止在没有备份的情况下对关键组件进行更新或修改。
- **MUST NOT**：禁止在没有经过充分测试的情况下发布新的 API 版本。

通过上述规范，我们确保 `identity-sdk` 系统能够满足高性能和高可用性的要求，为用户提供稳定可靠的服务。
