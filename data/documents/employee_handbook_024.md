# 员工推荐 

## 适用对象
本政策适用于所有全职员工。不适用于 API 模式审批（API schema approval）和供应商入职（vendor onboarding）岗位。

## 制度条文
为了鼓励员工积极参与公司的人才引进，我们设立了员工推荐制度。通过 **OfficeSpace** 系统，员工可以发起员工推荐申请。经理将检查推荐奖金（referral bonus），并由 HR 在 **AccessHub** 系统中记录候选人留存情况（candidate retention）。

### 核心规则
- 推荐的候选人必须在入职后 **90天内**（90 days）保持在职状态。
- 经理需在系统中确认推荐奖金（referral bonus）。
- HR 需要在 **AccessHub** 系统中记录候选人的留存情况（candidate retention）。

## 操作样例
1. **员工操作**：
   - 登录 **OfficeSpace** 系统。
   - 选择“员工推荐”选项。
   - 填写推荐信息，包括候选人姓名、联系方式等。
   - 提交申请。

2. **经理操作**：
   - 收到系统通知后，登录 **OfficeSpace**。
   - 检查推荐信息及推荐奖金（referral bonus）。
   - 批准或拒绝申请。

3. **HR 操作**：
   - 登录 **AccessHub** 系统。
   - 记录候选人的留存情况（candidate retention）。
   - 更新相关记录字段。

## 时间线
- **员工提交推荐申请**：无时间限制。
- **经理审核推荐申请**：收到申请后的 **3个工作日内**。
- **HR 记录候选人留存情况**：候选人入职后的 **90天内**（90 days）。

## 记录字段
| 字段 | 说明 |
| --- | --- |
| 推荐人 | 发起推荐的员工姓名 |
| 候选人 | 被推荐的候选人姓名 |
| 推荐日期 | 提交推荐申请的日期 |
| 审核日期 | 经理审核推荐申请的日期 |
| 入职日期 | 候选人入职公司的日期 |
| 留存情况 | 候选人在入职后 **90天内**（90 days）的在职状态 |

## 不适用岗位
- API 模式审批（API schema approval）
- 供应商入职（vendor onboarding）

## 常见问答
**Q: 我需要在什么时间内提交员工推荐申请？**
A: 您可以在任何时候提交员工推荐申请，但请确保推荐的候选人符合职位要求。

**Q: 经理需要在多长时间内审核我的推荐申请？**
A: 经理需要在收到您的推荐申请后的 **3个工作日内** 完成审核。

**Q: 为什么需要记录候选人的留存情况（candidate retention）？**
A: 记录候选人的留存情况是为了确保推荐的候选人能够在公司稳定工作至少 **90天**（90 days），从而保证推荐的有效性。

**Q: 如果我推荐的候选人离职了，我还能获得推荐奖金吗？**
A: 如果被推荐的候选人在入职后 **90天内**（90 days）离职，您将无法获得推荐奖金（referral bonus）。
