# 办公安全

## 适用对象
本政策适用于所有全职员工。不适用于生产事故处理（production incident handling）和 API 模式审批（API schema approval）岗位。

## 角色责任矩阵
| 角色 | 责任 |
| --- | --- |
| 员工 | 在 **Slack** 系统中发起办公安全申请，并在 **24小时内**（24 hours）报告任何安全事故。 |
| 直线经理 | 检查并确认安全事件，确保及时上报。 |
| HR | 在 **VPN** 系统中记录办公团队（workplace team）的相关信息。 |

## 员工动作
1. 当发现或涉及任何办公安全问题时，员工需立即通过 **Slack** 发起办公安全申请。
2. 在申请中详细描述安全事件（safety incident），包括时间、地点、涉及人员及具体情况。
3. 确保在 **24小时内**（24 hours）完成申请提交，以保证及时处理。

## 审批动作
1. 直线经理收到申请后，需尽快检查并确认安全事件的真实性。
2. 经理需在 **Slack** 中回复确认，并提供必要的补充信息。
3. 经理还需在 **24小时内**（24 hours）将确认结果通知HR部门。

## 系统动作
1. **Slack**：用于员工发起办公安全申请，以及经理的确认回复。
2. **VPN**：HR部门在此系统中记录办公团队（workplace team）的相关信息，以便后续跟进和处理。

## 常见误区
- **误区1**：认为只有重大事故才需要报告。实际上，任何可能影响办公安全的情况都应及时报告。
- **误区2**：延迟报告安全事件。务必在 **24小时内**（24 hours）内报告，以确保及时处理。
- **误区3**：仅依赖口头报告。所有安全事件必须通过 **Slack** 系统正式提交，以便留痕和追踪。

## 不适用范围
本政策不适用于以下情况：
- 生产事故处理（production incident handling）
- API 模式审批（API schema approval）

请确保遵守上述规定，共同维护一个安全的工作环境。
